Title: Bulk privatize non-essential repos

## Problem
Too many low-signal public repositories dilute portfolio quality and make senior-level review harder.

## Goal
Reduce public repo surface area by privatizing repos that do not support current positioning.

## Scope
- Review current public repositories
- Mark repos as keep public, privatize, or archive
- Exclude active portfolio repos and any repos needed for live demos

## Acceptance Criteria
- A reviewed inventory of current public repos exists
- Each repo has a keep/public/private decision
- Non-essential repos are queued for privatization
- Rationale is documented in one concise table or checklist

## Notes for Copilot coding agent
Keep this as a lightweight audit artifact. Do not automate repo visibility changes unless explicitly asked.

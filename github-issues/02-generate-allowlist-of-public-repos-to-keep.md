Title: Generate allowlist of public repos to keep

## Problem
There is no explicit allowlist of public repositories that should remain visible after cleanup.

## Goal
Create a short, intentional allowlist of repos that support senior/staff-level positioning.

## Scope
- Identify repos worth keeping public
- State why each repo earns a public slot
- Limit the list to high-signal work only

## Acceptance Criteria
- A concise allowlist exists
- Each kept repo has a one-line reason
- The list is short enough to review quickly
- The list can be used as the source of truth for visibility cleanup

## Notes for Copilot coding agent
Optimize for clarity, not volume. Prefer fewer strong repos over a long list.
